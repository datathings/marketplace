# Model Management API

## GET /api/tags -- List Local Models

List all models available locally.

### Request

No parameters. Simple GET request.

```bash
curl http://localhost:11434/api/tags
```

### Response

| Field | Type | Description |
|-------|------|-------------|
| `models` | ListModelResponse[] | Array of model entries |

Each model entry:

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Model name with tag |
| `model` | string | Model identifier |
| `remote_model` | string | Upstream model name (if applicable) |
| `remote_host` | string | Upstream host URL (if applicable) |
| `modified_at` | string | Last modification timestamp |
| `size` | int64 | Model size in bytes |
| `digest` | string | SHA256 digest |
| `details` | ModelDetails | Model metadata |

ModelDetails:

| Field | Type | Description |
|-------|------|-------------|
| `parent_model` | string | Parent model name |
| `format` | string | Format (e.g., `"gguf"`) |
| `family` | string | Model family (e.g., `"llama"`) |
| `families` | string[] | All model families |
| `parameter_size` | string | Parameter count (e.g., `"7.6B"`) |
| `quantization_level` | string | Quantization (e.g., `"Q4_K_M"`) |

## POST /api/show -- Show Model Information

Show details about a model including modelfile, template, parameters, license, and system prompt.

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | Yes | Model name |
| `system` | string | No | System message override |
| `template` | string | No | Template override (deprecated) |
| `verbose` | bool | No | Return full data for verbose fields (e.g., tokenizer data) |
| `options` | object | No | Runtime model parameters |

```bash
curl http://localhost:11434/api/show -d '{"model": "llava"}'
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `license` | string | Model license text |
| `modelfile` | string | Generated Modelfile content |
| `parameters` | string | Model parameters as key-value text |
| `template` | string | Prompt template |
| `system` | string | System prompt |
| `renderer` | string | Renderer used by the model |
| `parser` | string | Parser used by the model |
| `details` | ModelDetails | Model metadata (same as list endpoint) |
| `messages` | Message[] | Predefined messages |
| `model_info` | object | Raw model metadata (architecture, vocab, etc.) |
| `projector_info` | object | Projector metadata (for multimodal models) |
| `tensors` | Tensor[] | Tensor metadata (name, type, shape) |
| `capabilities` | string[] | Model capabilities (e.g., `["completion", "vision"]`) |
| `modified_at` | string | Last modification timestamp |
| `remote_model` | string | Upstream model name |
| `remote_host` | string | Upstream host URL |
| `requires` | string | Minimum Ollama version required |

Tensor object:

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Tensor name |
| `type` | string | Data type |
| `shape` | uint64[] | Tensor dimensions |

### Example Response

```json
{
  "modelfile": "# Modelfile generated by \"ollama show\"...",
  "parameters": "num_keep  24\nstop  \"<|start_header_id|>\"...",
  "template": "{{ if .System }}<|start_header_id|>system...",
  "details": {
    "parent_model": "",
    "format": "gguf",
    "family": "llama",
    "families": ["llama"],
    "parameter_size": "8.0B",
    "quantization_level": "Q4_0"
  },
  "model_info": {
    "general.architecture": "llama",
    "general.parameter_count": 8030261248,
    "llama.context_length": 8192,
    "llama.embedding_length": 4096
  },
  "capabilities": ["completion", "vision"]
}
```

## POST /api/create -- Create a Model

Create a model from another model, a safetensors directory, or a GGUF file. Streams status by default.

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | Yes | Name for the new model |
| `from` | string | No | Source model name |
| `remote_host` | string | No | URL of the upstream Ollama API for the model |
| `files` | object | No | Map of filenames to SHA256 digests (for GGUF/safetensors) |
| `adapters` | object | No | Map of filenames to SHA256 digests for LoRA adapters |
| `template` | string | No | Prompt template |
| `license` | string or string[] | No | License text(s) |
| `system` | string | No | System prompt |
| `parameters` | object | No | Model parameters |
| `messages` | Message[] | No | Predefined conversation messages |
| `renderer` | string | No | Renderer for the model |
| `parser` | string | No | Parser for the model |
| `requires` | string | No | Minimum Ollama version required |
| `info` | object | No | Additional information for the model |
| `stream` | bool | No | Stream response (default: `true`) |
| `quantize` | string | No | Quantization type (`"q4_K_M"`, `"q4_K_S"`, `"q8_0"`) |

### Response (streaming)

Stream of JSON objects with a `status` field:

```json
{"status": "reading model metadata"}
{"status": "creating system layer"}
{"status": "writing manifest"}
{"status": "success"}
```

For quantization, progress is reported:
```json
{"status": "quantizing F16 model to Q4_K_M", "digest": "0", "total": 6433687776, "completed": 12302}
```

### Example: Create from Existing Model

```bash
curl http://localhost:11434/api/create -d '{
  "model": "mario",
  "from": "llama3.2",
  "system": "You are Mario from Super Mario Bros."
}'
```

### Example: Quantize a Model

```bash
curl http://localhost:11434/api/create -d '{
  "model": "llama3.2:quantized",
  "from": "llama3.2:3b-instruct-fp16",
  "quantize": "q4_K_M"
}'
```

### Example: Create from GGUF

First upload the GGUF file as a blob, then reference its digest:

```bash
curl http://localhost:11434/api/create -d '{
  "model": "my-gguf-model",
  "files": {
    "test.gguf": "sha256:432f310a77f4650a88d0fd59ecdd7cebed8d684bafea53cbff0473542964f0c3"
  }
}'
```

### Example: Create from Safetensors

```bash
curl http://localhost:11434/api/create -d '{
  "model": "fred",
  "files": {
    "config.json": "sha256:dd3443e529fb2290423a0c65c2d633e67b419d273f170259e27297219828e389",
    "tokenizer.json": "sha256:bbc1904d35169c542dffbe1f7589a5994ec7426d9e5b609d07bab876f32e97ab",
    "model.safetensors": "sha256:1ff795ff6a07e6a68085d206fb84417da2f083f68391c2843cd2b8ac6df8538f"
  }
}'
```

## POST /api/pull -- Pull a Model

Download a model from the Ollama registry. Cancelled pulls resume from where they left off.

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | Yes | Model name to pull |
| `insecure` | bool | No | Allow insecure connections (deprecated, ignored) |
| `stream` | bool | No | Stream response (default: `true`) |

```bash
curl http://localhost:11434/api/pull -d '{"model": "llama3.2"}'
```

### Response (streaming)

```json
{"status": "pulling manifest"}
{"status": "pulling digestname", "digest": "digestname", "total": 2142590208, "completed": 241970}
{"status": "verifying sha256 digest"}
{"status": "writing manifest"}
{"status": "removing any unused layers"}
{"status": "success"}
```

Non-streaming (`"stream": false`):
```json
{"status": "success"}
```

## POST /api/push -- Push a Model

Upload a model to a registry. Requires registration and a public key.

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | Yes | Model name as `<namespace>/<model>:<tag>` |
| `insecure` | bool | No | Allow insecure connections |
| `stream` | bool | No | Stream response (default: `true`) |

```bash
curl http://localhost:11434/api/push -d '{"model": "mattw/pygmalion:latest"}'
```

### Response (streaming)

```json
{"status": "retrieving manifest"}
{"status": "starting upload", "digest": "sha256:bc07c81...", "total": 1928429856}
{"status": "pushing manifest"}
{"status": "success"}
```

## POST /api/copy -- Copy a Model

Create a copy of a model with a new name.

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source` | string | Yes | Source model name |
| `destination` | string | Yes | Destination model name |

```bash
curl http://localhost:11434/api/copy -d '{
  "source": "llama3.2",
  "destination": "llama3-backup"
}'
```

### Response

- **200 OK** on success
- **404 Not Found** if source model does not exist

## GET /api/ps -- List Running Models

List models currently loaded into memory.

### Request

No parameters.

```bash
curl http://localhost:11434/api/ps
```

### Response

| Field | Type | Description |
|-------|------|-------------|
| `models` | ProcessModelResponse[] | Array of running models |

Each running model:

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Model name |
| `model` | string | Model identifier |
| `size` | int64 | Total model size in bytes |
| `digest` | string | SHA256 digest |
| `details` | ModelDetails | Model metadata |
| `expires_at` | string | When the model will be unloaded |
| `size_vram` | int64 | Size loaded in VRAM |
| `context_length` | int | Current context length |

### Example Response

```json
{
  "models": [
    {
      "name": "mistral:latest",
      "model": "mistral:latest",
      "size": 5137025024,
      "digest": "2ae6f6dd7a3dd734790bbbf58b8909a606e0e7e97e94b7604e0aa7ae4490e6d8",
      "details": {
        "parent_model": "",
        "format": "gguf",
        "family": "llama",
        "families": ["llama"],
        "parameter_size": "7.2B",
        "quantization_level": "Q4_0"
      },
      "expires_at": "2024-06-04T14:38:31.83753-07:00",
      "size_vram": 5137025024,
      "context_length": 8192
    }
  ]
}
```

## HEAD /api/blobs/:digest -- Check if Blob Exists

Check whether a file blob exists on the server.

### Path Parameters

| Parameter | Description |
|-----------|-------------|
| `digest` | SHA256 digest of the blob |

```bash
curl -I http://localhost:11434/api/blobs/sha256:29fdb92e57cf0827ded04ae6461b5931d01fa595843f55d36f5b275a52087dd2
```

### Response

- **200 OK** if the blob exists
- **404 Not Found** if it does not

---

## POST /api/blobs/:digest -- Upload a Blob

Push a file to the Ollama server to create a blob (Binary Large Object). Used before `/api/create` for GGUF and safetensors models.

### Path Parameters

| Parameter | Description |
|-----------|-------------|
| `digest` | Expected SHA256 digest of the file |

```bash
curl -T model.gguf -X POST http://localhost:11434/api/blobs/sha256:29fdb92e57cf0827ded04ae6461b5931d01fa595843f55d36f5b275a52087dd2
```

### Response

- **201 Created** if blob was successfully created
- **400 Bad Request** if the digest does not match
