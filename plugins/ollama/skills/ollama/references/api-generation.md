# Generation and Chat API

## POST /api/generate -- Text Generation

Generate a response for a given prompt. Streams by default.

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | Yes | Model name (e.g., `"llama3.2"`) |
| `prompt` | string | No | Text prompt to generate from |
| `suffix` | string | No | Text after model response (fill-in-the-middle) |
| `system` | string | No | System message (overrides Modelfile) |
| `template` | string | No | Prompt template (overrides Modelfile) |
| `context` | int[] | No | Context from previous `/api/generate` response (deprecated) |
| `stream` | bool | No | Stream response (default: `true`) |
| `raw` | bool | No | Skip prompt templating |
| `format` | string or object | No | `"json"` for JSON mode, or a JSON Schema object for structured output |
| `keep_alive` | string or number | No | How long to keep model loaded (default: `"5m"`) |
| `images` | string[] | No | Base64-encoded images for multimodal models |
| `options` | object | No | Runtime model parameters (temperature, top_k, etc.) |
| `think` | bool or string | No | Enable thinking: `true`/`false` or `"high"`/`"medium"`/`"low"` |
| `truncate` | bool | No | Truncate prompt if it exceeds context length |
| `shift` | bool | No | Shift context window instead of erroring on overflow |
| `logprobs` | bool | No | Return log probabilities of output tokens |
| `top_logprobs` | int | No | Number of top logprobs to return per token (0-20, requires `logprobs: true`) |

Experimental image generation fields (for image generation models only):

| Field | Type | Description |
|-------|------|-------------|
| `width` | int32 | Width of generated image in pixels |
| `height` | int32 | Height of generated image in pixels |
| `steps` | int32 | Number of diffusion steps |

### Response Fields (streaming)

Each streamed JSON object contains:

| Field | Type | Description |
|-------|------|-------------|
| `model` | string | Model name |
| `remote_model` | string | Upstream model name (if applicable) |
| `remote_host` | string | Upstream host URL (if applicable) |
| `created_at` | string | Timestamp (RFC3339) |
| `response` | string | Generated text token (empty in final response if streamed) |
| `thinking` | string | Thinking/reasoning content (when `think` is enabled) |
| `done` | bool | Whether generation is complete |
| `done_reason` | string | Reason for stopping (e.g., `"stop"`, `"unload"`, `"load"`) |
| `tool_calls` | ToolCall[] | Tool calls generated by the model (if applicable) |
| `logprobs` | array | Log probability info (if requested) |

Experimental image generation response fields (for image generation models only):

| Field | Type | Description |
|-------|------|-------------|
| `image` | string | Base64-encoded generated image |
| `completed` | int64 | Number of completed diffusion steps (during streaming) |
| `total` | int64 | Total number of diffusion steps (during streaming) |

The final response (`done: true`) additionally includes:

| Field | Type | Description |
|-------|------|-------------|
| `context` | int[] | Conversation context encoding (for follow-up requests) |
| `total_duration` | int | Total time (nanoseconds) |
| `load_duration` | int | Model loading time (nanoseconds) |
| `prompt_eval_count` | int | Tokens in prompt |
| `prompt_eval_duration` | int | Prompt evaluation time (nanoseconds) |
| `eval_count` | int | Tokens generated |
| `eval_duration` | int | Generation time (nanoseconds) |

### Example: Streaming

```bash
curl http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "Why is the sky blue?"
}'
```

Streamed token:
```json
{
  "model": "llama3.2",
  "created_at": "2023-08-04T08:52:19.385406455-07:00",
  "response": "The",
  "done": false
}
```

```json
{
  "model": "llama3.2",
  "created_at": "2023-08-04T19:22:45.499127Z",
  "response": "",
  "done": true,
  "context": [1, 2, 3],
  "total_duration": 10706818083,
  "load_duration": 6338219291,
  "prompt_eval_count": 26,
  "prompt_eval_duration": 130079000,
  "eval_count": 259,
  "eval_duration": 4232710000
}
```

### Example: Raw Mode (No Templating)


### Example: Load/Unload Model

Load a model into memory (empty prompt):
```bash
curl http://localhost:11434/api/generate -d '{"model": "llama3.2"}'
```

Unload a model from memory:
```bash
curl http://localhost:11434/api/generate -d '{"model": "llama3.2", "keep_alive": 0}'
```

## POST /api/chat -- Chat Completion

Generate the next message in a chat conversation. Streams by default.

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | Yes | Model name |
| `messages` | Message[] | Yes | Conversation messages |
| `tools` | Tool[] | No | Available tools for the model |
| `think` | bool or string | No | Enable thinking: `true`/`false` or `"high"`/`"medium"`/`"low"` |
| `format` | string or object | No | `"json"` or JSON Schema for structured output |
| `options` | object | No | Runtime model parameters |
| `stream` | bool | No | Stream response (default: `true`) |
| `keep_alive` | string or number | No | How long to keep model loaded (default: `"5m"`) |
| `truncate` | bool | No | Truncate prompt if it exceeds context length |
| `shift` | bool | No | Shift context window instead of erroring on overflow |
| `logprobs` | bool | No | Return log probabilities |
| `top_logprobs` | int | No | Number of top logprobs per token (0-20) |

### Message Object

| Field | Type | Description |
|-------|------|-------------|
| `role` | string | `"system"`, `"user"`, `"assistant"`, or `"tool"` |
| `content` | string | Message text |
| `thinking` | string | Model's thinking text (for thinking models) |
| `images` | string[] | Base64-encoded images (for multimodal models) |
| `tool_calls` | ToolCall[] | Tool calls made by the assistant |
| `tool_name` | string | Name of tool that produced this result (role=tool) |
| `tool_call_id` | string | ID of the tool call being responded to |

### ToolCall Object

```json
{
  "id": "optional-id",
  "function": {
    "index": 0,
    "name": "function_name",
    "arguments": {"key": "value"}
  }
}
```

### Tool Object

```json
{
  "type": "function",
  "function": {
    "name": "tool_name",
    "description": "What the tool does",
    "parameters": {
      "type": "object",
      "properties": {
        "param_name": {
          "type": "string",
          "description": "Parameter description"
        }
      },
      "required": ["param_name"]
    }
  }
}
```

### Response Fields (streaming)

Each streamed object:

| Field | Type | Description |
|-------|------|-------------|
| `model` | string | Model name |
| `remote_model` | string | Upstream model name (if applicable) |
| `remote_host` | string | Upstream host URL (if applicable) |
| `created_at` | string | Timestamp |
| `message` | Message | Response message (partial content during streaming) |
| `done` | bool | Whether generation is complete |
| `done_reason` | string | Reason for stopping |
| `logprobs` | array | Log probability info (if requested) |

Final response (`done: true`) additionally includes metrics:

| Field | Type | Description |
|-------|------|-------------|
| `total_duration` | int | Total time (nanoseconds) |
| `load_duration` | int | Model loading time (nanoseconds) |
| `prompt_eval_count` | int | Tokens in prompt |
| `prompt_eval_duration` | int | Prompt evaluation time (nanoseconds) |
| `eval_count` | int | Tokens generated |
| `eval_duration` | int | Generation time (nanoseconds) |
