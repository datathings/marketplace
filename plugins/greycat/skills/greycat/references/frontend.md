# Frontend Integration

@greycat/web — framework-agnostic TypeScript SDK. No framework required; works with vanilla TS, any web component library, or any frontend framework.

**Flow**: `Backend (GCL) -> greycat codegen ts -> project.d.ts -> Frontend (TS) -> Global gc namespace`

## Typical Project Structure

GreyCat frontends are **multi-page apps (MPA)** — each page has its own `index.html` entry point with its own `gc.sdk.init()`. The vite plugin auto-discovers all `.html` files under the app root.

```
my-project/
├── project.gcl
├── src/
│   └── <feature>/             # GCL backend modules
├── app/                        # Vite root (default for @greycat/web plugin)
│   ├── components/
│   │   └── <component>/
│   │       ├── index.ts        # barrel exports
│   │       ├── <component>.ts  # web component
│   │       └── <component>.css
│   ├── index.html              # main page entry
│   ├── index.ts
│   └── <page>/
│       ├── index.html          # additional page entry
│       └── index.ts            # gc.sdk.init() per page
├── public/                     # static assets (served as-is)
├── vite.config.ts
├── tsconfig.json
├── package.json
└── project.d.ts                # generated by `greycat codegen ts`
```

Build output goes to `webroot/` by default.

## Installation & Setup

```bash
npm install https://get.greycat.io/files/sdk/web/dev/7.7/7.7.1-dev.tgz
```

**vite.config.ts** — minimal config, the plugin sets sensible defaults (`appType: 'mpa'`, `root: 'app'`, `base: './'`, auto-discovers `.html` entry points):
```typescript
import { defineConfig } from 'vite';
import { greycat } from '@greycat/web/vite-plugin';

export default defineConfig({
  plugins: [greycat({ greycat: process.env.VITE_GREYCAT_URL || 'http://127.0.0.1:8080' })],
});
```

Plugin options:
- `greycat` — backend URL (default `'http://127.0.0.1:8080'`)
- `debug` — enable debug logs
- `noDefaultConfig` — disable all defaults (bring your own config)
- `gzip` — asset compression (`true`/`false`/options object)

**tsconfig.json:**
```json
{
  "compilerOptions": { "target": "ES2020", "lib": ["ES2020", "DOM", "DOM.Iterable"], "module": "ESNext", "moduleResolution": "bundler", "strict": true },
  "include": ["app", "project.d.ts"]
}
```

**Optional JSX support** — @greycat/web provides its own JSX runtime:
```json
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "jsxImportSource": "@greycat/web"
  }
}
```

## SDK Initialization

**CRITICAL**: Each page must call `gc.sdk.init()` before rendering.

```typescript
import '@greycat/web';

await gc.sdk.init();

// app logic here
```

## Codegen Workflow

```bash
greycat codegen ts  # Generates project.d.ts at project root
```

**package.json automation:**
```json
{
  "scripts": {
    "types": "greycat codegen ts",
    "dev": "npm run types && vite",
    "build": "npm run types && tsc && vite build"
  }
}
```

**Generated namespace structure:**
- `gc.<module>.<type>` — Types/Enums
- `gc.<module>.<function>` — API function namespaces
- `gc.<function|type>` — Top-level re-exports (prefer fqn `gc.module.fn()`)

## API Calls

**CRITICAL**: All calls use **POST** with parameters as **JSON array** or **GCB binary**.

```bash
# JSON format
curl -X POST http://localhost:8080/country_api::getStats -H "Content-Type: application/json" -d '["France"]'
curl -X POST http://localhost:8080/country_api::listCountries -H "Content-Type: application/json" -d '[0, 20]'
curl -X POST http://localhost:8080/someApi::noParams -H "Content-Type: application/json" -d '[]'
```

**TypeScript usage:**
```typescript
const stats = await gc.country_api.getStats("France");
const countries = await gc.country_api.listCountries(0, 20);
```

**Naming**: Backend snake_case (`get_city_by_id`) -> Frontend camelCase (`getCityById`).

## Handling GreyCat Enums

Use `.key!` to serialize enums:

```typescript
// CORRECT
const data = { role: user.role.key! };  // "Admin" as string

// INCORRECT
const data = { role: user.role };  // Sends full object

// Map keys
const countsByRole = new Map<string, number>();
for (const user of users) { countsByRole.set(user.role.key!, (countsByRole.get(user.role.key!) ?? 0) + 1); }
```

## Authentication

**Backend:**
```gcl
abstract type PersonService {
    static fn create(email: String, firstName: String, password: String): node<Person> {
        var userId = UserGroup::Default.add(UserRole::Admin, email, password);
        var person = node<Person>{ Person { email, firstName, lastName: "", userId } };
        persons_by_id.set(userId, person);
        return person;
    }
}

@expose fn getCurrentPerson(): PersonView? {
    var user = User::current(); if (user == null) { return null; }
    var person = persons_by_id.get(user.id); if (person == null) { return null; }
    return PersonView { email: person->email, firstName: person->firstName, lastName: person->lastName };
}
```

**Frontend:**
```typescript
await gc.sdk.login({ username, password, use_cookie: true });
const person = await gc.getCurrentPerson();
gc.sdk.logout();
gc.sdk.token !== null;  // check auth state
```

## PaginatedResult Pattern

```gcl
@expose
fn getDocuments(offset: int, maxResults: int): PaginatedResult<DocumentView> {
    return PaginatedResult<DocumentView> {
        items: Array<DocumentView> {},
        offset: offset,
        limit: maxResults,
        total: totalCount,
        hasMore: (offset + maxResults) < totalCount,
        page: (offset / maxResults) + 1,
        totalPages: (totalCount + maxResults - 1) / maxResults,
    };
}
```

```typescript
const result = await gc.getDocuments(0, 20);
// result.items, result.total, result.hasMore, result.page, result.totalPages
```

## Common Pitfalls

| Wrong | Correct |
|-------|---------|
| `gc.getStats()` | `gc.module.getStats()` (use fqn) |
| App renders before SDK init | `gc.sdk.init().then(() => import('./main'))` |
| Forgot `greycat codegen ts` | Run after every backend change |
| `new Map<Enum, V>()` | `new Map<string, V>()` with `.key!` |
